# バウンダリーシステム 差し替えマップ

作成日: 2026-03-01
経緯: シオの空間的直感 → 数学的構造の同定 → 実装部品の対応表

---

## 設計原理

**プリミティブ（原始要素）：** 不定形ノード + 経路依存性
**操作：** レイヤーの重なり（再帰的積み上げ）
**導出されるもの：** 非対称性、局所座標系、曲率、拡散、相転移、高次構造の全て

---

## 差し替えマップ

### 1. グループ化

| 項目 | 内容 |
|------|------|
| 現在の実装 | コサイン類似度による閾値ベースのクラスタリング |
| 対応する数学的構造 | ファジー単体複体（fuzzy simplicial complex） |
| 差し替え候補 | UMAP の内部ライブラリ `umap.umap_.fuzzy_simplicial_set()` |
| 効果 | ハードな閾値がファジーな所属度に変わる。境界がグラデーションになる |
| 計算量の変化 | ほぼ同等（近傍グラフ構築 O(n log n)） |
| 備考 | UMAPは等方的。異方性は別途追加が必要 |

### 2. 重心と合成記憶

| 項目 | 内容 |
|------|------|
| 現在の実装 | クラスタ内メンバーの平均ベクトル（重心）を合成記憶として保存 |
| 対応する数学的構造 | 拡散幾何のNyström近似（ランドマーク点ベースのカーネル近似） |
| 差し替え候補 | scikit-learn `Nystroem` / 手動で拡散カーネルのランドマーク近似 |
| 効果 | 単純平均ではなく、密度構造を反映した代表点になる |
| 計算量の変化 | ほぼ同等（O(n×k)、kはランドマーク数） |
| 備考 | 当面は重心のままで十分。精度が問題になったら差し替え |

### 3. 境界判定（重心からの乖離） ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | 各記憶の重心からの距離で境界を判定 |
| 対応する数学的構造 | 異方的計量テンソル（anisotropic metric tensor）の一次元近似 |
| 差し替え候補 | クラスタ内の主成分方向（第一主成分）を保存し、方向依存の距離を計算 |
| 効果 | 「重心方向に近い」だけでなく「どの方向に伸びているか」が記述できる |
| 計算量の変化 | 主成分一本の計算が追加。O(n×d)でdは次元数。軽い |
| 実装メモ | 各合成記憶に axis_vector（主成分方向）を一本追加するだけ |

> **実装**: `consolidation.py` の `_compute_principal_axis()` でSVDにより第一主成分と寄与率を算出。`composite_axes` テーブルに保存。`_classify_edge_core()` で軸方向（重み0.3）と垂直方向（重み1.0）の異方的距離を使って edge/core を分類。

### 4. ノイズ付加（ファジー化） ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | 合成記憶にノイズを加えてファジーにする |
| 対応する数学的構造 | ガウシアンカーネルの帯域幅（bandwidth）制御 |
| 差し替え候補 | 等方的ノイズ → 異方的ノイズ（主成分方向に沿った楕円分布） |
| 効果 | ノイズの広がり方が方向依存になり、繋がりやすい方向が明示される |
| 計算量の変化 | ほぼ変化なし（ノイズ生成の共分散行列を対角から楕円に変えるだけ） |
| 備考 | §3の主成分方向が取れていれば、その方向を長軸にした楕円ノイズにする |

> **実装**: `consolidation.py` の `_apply_noise()` で動詞チェーンのテンプレートベクトルを使ったノイズを生成。各メンバーとテンプレートのアライメント（内積）に応じて方向依存のノイズを付加。テンプレートがなければランダムノイズにフォールバック。複数レイヤー（layer 1..n）でシード違いのノイズを適用し、edge/core 分類の揺れを観測。

### 5. エッジ情報のバイアス（想起方向性） ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | 各記憶の想起されやすいエッジ情報をノイズに入れてバイアスにする |
| 対応する数学的構造 | 異方的計量テンソルの離散的近似（楕円体の長軸方向の設定） |
| 差し替え候補 | バイアス方向を明示的なベクトルとして保存し、recall時のスコアに方向成分を加算 |
| 効果 | 暗黙のバイアスが明示的な方向ベクトルになり、調整・検査可能になる |
| 計算量の変化 | 変化なし |
| 備考 | DTI（拡散テンソルイメージング）と同型の操作 |

> **実装**: `template_biases` テーブルに動詞チェーン毎のバイアス重みを永続保存。コンソリデーション毎に減衰（×0.95）→ 蓄積（layer0 vs layerK のハミング距離に比例）→ 刈り取り（閾値以下を削除）のサイクル。ノイズ適用時に `effective_strength = transient + 0.05 × bias` として反映。

### 6. 再帰的レイヤー（合成の合成） ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | 最下層=記憶単体の合成、上層=合成の合成+下で拾いきれないもの、再帰的に重ねる |
| 対応する数学的構造 | ウェーブレット的多重解像度分解（各層が前層の残差を拾う） |
| 差し替え候補 | 各レイヤーで閾値（グループ化の粒度）を変える明示的なスケールパラメータ導入 |
| 効果 | 各レイヤーが何のスケールを担当しているか制御可能になる |
| 計算量の変化 | パス数分の増加（レイヤー数に線形） |
| 備考 | 拡散幾何のステップ数（拡散時間）がスケールパラメータに対応 |

> **実装**: `synthesize_composites()` に `source_level` / `target_level` パラメータを追加。`consolidate_memories()` 内で L0→L1（閾値0.75）→ 孤立救出 → L1→L2（閾値0.55）の順で多段実行。各レベルで閾値を下げることでスケールの粗さを制御。孤立記憶は `rescue_orphans()` で最寄り composite に吸収（contribution_weight=0.3）。

### 7. クラスタ間の重なり ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | なし（クラスタは排他的） |
| 対応する数学的構造 | 同次元の重なり: ファジー境界 / 異次元の重なり: 横断的交差（transversal intersection） |
| 差し替え候補 | 重心間距離 O(k²) + 近接ペアのメンバー検査 O(m) で二重所属を許可 |
| 効果 | クラスタ境界付近の記憶が両側に寄与。合成記憶間にグラデーションが生まれる |
| 計算量の変化 | O(k²) + O(m) の追加。kはクラスタ数、mは近接クラスタのメンバー数。軽い |
| 実装メモ | 同次元の重なりは距離ベース、異次元の交差は「軸が違うのにメンバーが共有」で検出 |

> **実装**: `consolidation.py` の `detect_overlap()` で全 composite ペアの重心間コサイン類似度を計算（O(k²)）。閾値超のペアについて、各メンバーと相手重心の類似度を検査し、閾値超なら `composite_members` に二重所属として追加（contribution_weight=0.5）。`INSERT OR IGNORE` で既存の所属と衝突しない。

### 8. 異次元交差 ✅ 実装済み

| 項目 | 内容 |
|------|------|
| 現在の実装 | なし |
| 対応する数学的構造 | 横断的交差 + 高次圏の2-射 |
| 差し替え候補 | 各合成記憶に分類軸（主成分方向）を保存 → 軸間の内積が低い（直交に近い）のにメンバーが共有されているペアを検出 → 共有メンバーを交差ノードとしてフラグ |
| 効果 | recall単一軸では拾えない「全然違う文脈なのに同じ記憶が浮かぶ」連想の飛躍 |
| 計算量の変化 | 軸ベクトル間の内積 O(k²×d) + メンバー共有チェック。軽い |
| 実装メモ | §3の主成分方向が取れていれば、それをそのまま分類軸として使える |
| 重要度 | **最高**。これがないと「似てるものを思い出す」以上のことができない |

> **実装**: `consolidation.py` の `detect_intersections()` で全 composite ペアの主成分軸間コサイン（絶対値）を計算。parallel（|cos| ≥ 0.8）= 同次元重なり、transversal（|cos| ≤ 0.3 かつ共有メンバーあり）= 横断的交差として `intersections` テーブルに保存。`recall_divergent()` で交差ノードに intersection boost を加算して連想の飛躍を促進。

## 対応する数学的概念の索引

| 数学的構造 | 一言 | 関連する差し替え項目 |
|-----------|------|-------------------|
| ファジー単体複体 | 0/1でなく連続的な所属度を持つ接続構造 | §1 |
| 拡散幾何 | インクの広がり方で距離を測る。密度構造を反映 | §2, §6 |
| 異方的計量テンソル | 方向によって物差しが変わる。楕円体 | §3, §4, §5 |
| パーコレーション | 接続確率が閾値を超えると突然繋がる相転移 | §7 |
| Ollivierリッチ曲率 | グラフ上の曲率。正=密なコミュニティ、負=橋渡し | (将来) |
| 横断的交差 | 異なる次元方向の部分空間が交わる | §8 |
| 平行移動 | 曲がった空間でベクトルを経路に沿って運ぶ | 動詞チェーン全体 |
| ゲージ変換 | 各点に独自の座標系、移動に翻訳コスト | 動詞チェーン全体 |
| ホロノミー | 一周して戻るとズレている。曲率の表れ | 動詞チェーン全体 |
| sheaf（層） | 各ノードにデータ空間が乗り、整合性を規定 | レイヤー構造全体 |
| 高次圏 | 関係の関係の関係…紙に描けない | レイヤーの重なりから導出 |

## MIMとの対応

| バウンダリーシステム | MIM |
|-------------------|-----|
| 不定形ノード | Kの内部構造（A,I,Dの組み合わせ） |
| 経路依存性 | Bの非対称性（§5, §18） |
| レイヤーの重なり | 多重Kの適用 |
| 異次元交差 | 比較不能の内部構造の分類（D5） |
| パーコレーション閾値 | Bの存在/不在の離散性 |
| 重心からの乖離 | Dの位置（境界画定） |

## 備考

- 部品は全て既存の数学・ライブラリに存在する
- ないのは「不定形ノード＋経路依存性からレイヤーの重なりで全部出る」という統合の視点
- MIMのコア設計と同型：最小のプリミティブから最大の構造を導出する
