# 記憶システム 

実装済み概念

---

## 1. 意味は経路

体験を**動詞チェーン**で記録する理由。
「猫を撫でて安心した」と「犬を撫でて安心した」。名詞は違うけど、人間は「似てる」と感じる。似てるのは猫と犬じゃなくて、「撫でる → 安心する」という流れの方。

```
見る(空) → 気になる(色) → 調べる(天気) → 理解する(理由)
```

ベクトル化するとき、2つの軸に分離する：

- **flow_vector**: 動詞バイグラム（隣接動詞ペアの中点）の平均。「何をしたか」
- **delta_vector**: 名詞平均 − 動詞平均。「何に対してか」

chiVe（日本語word2vec, 300次元）で各単語をベクトル化し、上記の計算で2本のベクトルを得る。検索時は2軸を独立に比較する（flow=0.6, delta=0.4）。

この分離により、「同じことを違うものにした」（flow平行 × delta直交）と「違うことを同じものにした」（flow直交 × delta平行）を区別できる。

---

## 2. 記憶は点ではなく領域

一つの記憶をベクトル1本に潰すと形が消える。「猫を撫でた → 安心した → 子供の頃を思い出した」は点ではなく、連する要素が作る集合。その集合は不定形。

形の表現：

- **合成記憶**: 似た記憶をグループ化して、重心だけじゃなく全メンバーを保持
- **edge/core分類**: グループ内で中心にいるか外縁にいるかを分ける。外縁が境界
- **多層ノイズ**: 動詞チェーンのパターンをノイズとして加える。バイアスの近似

---

## 3. 空間は異方的

**同じ経路でも出発点が違えば見え方が変わる。**

「猫を撫でた → 安心した」と「猫を撫でた → 不安になった」、違う方向。逆に「猫を撫でた → 安心した」と「雨音を聞いた → 安心した」、名詞も出発点も違うが似た形。


- **主成分軸**: 各合成記憶のSVD第一主成分を保存。クラスタが一番伸びてる方向。楕円体
- **異方的距離**: 軸方向の距離と軸に垂直な距離で重みが違う。軸方向に遠いのは「まだ中」、垂直方向に同じだけ遠いのは「外」
- **方向依存ノイズ**: 経験の多い方向には境界が柔らかく、未経験の方向には硬い

距離を非対称に。記憶Aから記憶Bへは近道があっても、BからAへは遠回りかもしれない。「猫から安心を思い出す」と「安心から猫を思い出す」が同じ強さとは限らない。連想に方向がある。

---

## 4. マルチモーダル拡張

**各モダリティは独立した空間を持ち、対応を前提にしない。**

CLIPやImageBindは大量のデータで「テキストと画像の対応」を事前に学習し、統一された空間を作る。このシステムはそれを前提としない。各モダリティ（テキスト/画像/音声）は独立したエンコーダと独立したベクトル空間を持つ。

### 実装: ベクトル空間 + グラフによる橋渡し

実際の実装では、バイアス/バウンダリーの相互依存ではなく、**ベクトル空間とグラフの役割分担**で橋渡しを行う。

#### 画像→ベクトル空間（ふわっとした類似度）

vision-server（MobileCLIP + MediaPipe）が画像をセグメント→ベクトル化する。

```
画像 → MediaPipeセグメンテーション → 前景(人物) / 背景(場所) / 顔 に分離
     → MobileCLIPで各セグメントをベクトル化
     → image_embeddings テーブル（独立テーブル、memoriesとは別）
```

| ベクトル | 対象 | 用途 |
|---------|------|------|
| flow_vector | 背景（人物除去後） | 場所の記憶 |
| delta_vector | 人物セグメント | 人物の記憶 |
| face_vector | 顔クロップ | 顔の記憶 |

テキスト記憶のflow/deltaとは独立した空間（chiVe 300次元 vs MobileCLIP 512次元）。同じ名前を使っているが、空間は共有しない。

#### ベクトル空間→グラフ（離散的な名前ノード）

image_composites（同一人物のクラスタ）にtag（名前）を付与すると、`consolidate_memories` 時に `strengthen_visual_graph_edges()` がグラフノードに橋渡しする。

```
image_composites (tag="シオ") → graph_nodes: 「見る → シオ」vn edge
                                              「いる → シオ」vn edge（背景composite）
```

エッジの重みは `0.2 × member_count / max_member_count`（よく見るほど強い）。

#### テキスト記憶↔視覚記憶の接続

ベクトル空間（ふわっとした重心でI2I類似度を測る）とグラフ（離散的な名前ノードでテキスト記憶と繋がる）が役割分担する。

- **ベクトル空間**: 「この人、前に見た人に似てる」（画像間の類似度）
- **グラフ**: 「見た→シオ」（名前付きの関係性、テキスト記憶空間への入口）

テキストと画像を統一ベクトル空間に射影するのではなく、グラフの名前ノードを介して繋がる。

### 対応できないものは対応しない

別モダリティの軸はテキスト軸に完全には対応しない。対応できるものがあってもそれは偶然。このシステムは対応を前提にしないので、各軸が独自の情報を持ったまま残る。言葉にできない視覚の記憶は、言葉にできないまま保持される。
